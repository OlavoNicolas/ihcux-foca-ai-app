# FocaAí

### Interação Humano Computador e UX
Professor: Daniel Henrique Matos de Paiva

# Sobre o Projeto
O projeto FocaAí foi desenvolvido como atividade prática da disciplina de Interação Humano Computador e UX, com foco na criação de um protótipo de baixa fidelidade utilizando o Miro.
A proposta do aplicativo é auxiliar estudantes e profissionais a manterem o foco durante estudos e trabalho, reduzindo distrações e incentivando hábitos mais saudáveis no uso do celular.

# Funcionalidades do Aplicativo
O protótipo apresenta as seguintes funcionalidades principais:

* Modo Foco (Pomodoro)
* Bloqueio seletivo de aplicativos
* Jardim Digital com crescimento de árvore virtual
* Estatísticas de tempo ganho
* Desafios em grupo para sessões coletivas de foco

# Conceito de Design
A interface foi projetada com base nos princípios de Calm Technology, priorizando uma experiência mais tranquila, intuitiva e menos estimulante.

Para reduzir ansiedade e evitar sobrecarga visual, utilizamos:

* Espaço em branco para transmitir sensação de calma;
* Poucos elementos por tela;
* Hierarquia visual clara, destacando apenas o essencial;
* Tipografia simples e legível;
* Feedbacks discretos e não invasivos;
* Cores suaves e minimalistas;
* Navegação simples e direta.

O cronômetro foi definido como elemento central da interface para facilitar a concentração do usuário sem distrações secundárias.

# Fluxo de Usuário
O fluxo principal para iniciar uma sessão de foco em grupo acontece da seguinte maneira:

1. O usuário acessa a Dashboard Principal;
2. Define o tempo de foco utilizando o seletor;
3. Escolhe um ambiente sonoro relaxante;
4. Configura quais aplicativos serão bloqueados;
5. Entra ou cria uma sala de foco em grupo;
6. Inicia a sessão de concentração;
7. Durante o foco, a árvore virtual cresce conforme o usuário permanece no aplicativo;
8. Caso o usuário tente sair antes do tempo, uma tela de aviso é exibida;
9. Ao final da sessão, o usuário visualiza estatísticas e progresso.

# Telas Desenvolvidas
O protótipo possui as seguintes telas principais:

1. Dashboard Principal

4. Ranking do Grupo
5. Relatório de Conquistas
6. Tela de Tentativa de Saída

# Escolha de Componentes
## Slider para definição do tempo

Foi utilizado um slider em vez de digitação numérica para tornar a interação mais rápida, intuitiva e menos cansativa. O componente também reduz erros de preenchimento.

## Interruptores On/Off
Os interruptores foram utilizados na configuração de bloqueio por serem facilmente reconhecidos pelos usuários e transmitirem claramente estados ativos e inativos.

## Cartões Minimalistas
As informações foram organizadas em cartões simples para melhorar leitura e separação visual sem gerar excesso de estímulos.

## Indicadores Visuais de Progresso
O crescimento da árvore virtual e os gráficos de foco ajudam o usuário a perceber evolução sem utilizar notificações agressivas.

# Reflexão de UX
O objetivo do protótipo é incentivar o usuário a utilizar o celular de forma mais consciente. A interface foi desenhada para minimizar distrações e tornar o aplicativo um apoio silencioso durante momentos de concentração.
A proposta central é que a tecnologia ajude o usuário a focar na vida real, e não a permanecer preso à tela.
